# MIGRATION — walshero is home
*Founder ruling 2026-07-11. Asked for repeatedly. Treated as a settings change. It was not.*

## THE PROBLEM

**The institution owned the studio's operating system.**

Every governance file — the Command Center, the cross-lane manifest, the pre-ship gate,
eleven OS blocks, the Drive Atlas, the Harvest Engine, the Service Bay, and the folder
literally named `Tight Spiral Studio` — was owned by **mwalsh@post.massbay.edu**.

Not *shared with*. **Owned by.**

They landed that way because the Zapier connector was authed as the post account when
Claude created them. Nobody chose it. It accreted.

**The risk:** MassBay is an employer, not a partner. If that account is disabled —
offboarding, an IT policy change, a password reset outside the founder's control — the
operating layer of a personal business goes with it.

## THE FIX — not what it looks like

The instinct is "transfer ownership post → walshero." **Google usually forbids that across
domains.** Institutional Workspace configs block transfer to outside accounts. The obvious
move may not even be available.

**The real fix is deeper: CANON LEAVES DRIVE.**

| what | old home | NEW HOME |
|---|---|---|
| Governance (OS, manifest, gate, blocks) | Drive (post-owned) | **REPO** |
| Products (games, tools) | scattered | **REPO** |
| Live state (Command Center) | Drive | Drive — **walshero-owned** |
| Binary source (docx/xlsx/png) | Drive | Drive — **walshero-owned** |
| Anything >50KB | — | **REPO** + Drive pointer |

**The repo was always walshero's.** The migration is not moving Drive accounts — it is
**recognizing that Drive was never the right home for canon.** The ownership problem is
the proof.
