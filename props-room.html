The Props Room

Reusable single-file parts. Drop-in ready, floor-verified. First part: the Carry-Out.

The Carry-Out is the studio's persistence pattern. A single offline file can't store anything across a session boundary — so instead of storing, it emits a structured record the human carries out to a durable place (a Doc, an email, the Project). This loosens both persistence floors with one shape: the studio emits a roll-up; a game emits a player record. Don't store — emit and carry. The human stays the principal.

The source — paste this into any build

Generic carry-out emitter. Give it a title and a list of [label, value] pairs; it returns clean text. Works for a player session OR a studio session (just change the fields). No storage, no network — emits text only.

// CARRY-OUT — Tight Spiral Props Room v1. Offline, no storage, emits text only. // usage: carryOut("EN195 — Week 6", [["Got done", did],["Next thing", next]]) function carryOut(title, pairs){ var L = [title, ""]; pairs.forEach(function(p){ var label = p[0], val = (p[1]||"").toString().trim(); if(val) L.push(label + ": " + val); }); L.push(""); L.push("(I chose to share this. Nothing was sent automatically.)"); return L.join("\n"); } // copy-to-clipboard helper (clipboard is local; allowed under the offline floor) function copyOut(text, onDone){ if(navigator.clipboard){ navigator.clipboard.writeText(text).then(onDone, onDone); } else { var t=document.createElement("textarea"); t.value=text; document.body.appendChild(t); t.select(); try{document.execCommand("copy");}catch(e){} document.body.removeChild(t); if(onDone) onDone(); } }

Two costumes, one part

Player carry-out — ends a game; the player pastes their record to a Doc/email, or to the hub when it exists. Honest "learn as it goes": the player carries the evidence.

Studio carry-out — ends a session; the founder phone-saves the roll-up + decision tree to the Project. Memory becomes a cache; the carry-out is the system of record. The ceiling can't break what doesn't depend on memory.
