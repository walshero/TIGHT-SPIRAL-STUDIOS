Step 1 of 4: Headwaters — describe the assignment.

Start at the headwaters

Drop in an assignment. The river follows your own logic — what it teaches, how you assess it, which rubric — and lines up the gears before any work is scored. Fine-tune the assessment here, at the source, where it's cheap to change.

Question 1 — the artifact

The task as written. The Auditor reads this as the student artifact's source — the center of every ring.

Question 2 — what is it teaching?

The learning outcome in your own words. One clear sentence is enough.

Question 3 — how will you assess it?

This MVP lines up against the AAC&U Written Communication VALUE rubric. More dimensions are coming; this one ships first.

Nothing leaves this page. No network, no storage, no student data retained — the Auditor runs entirely in memory and forgets everything when you close it.

The run — the OS, shown not hidden

Your assignment flows through the Tight Spiral pipeline. Every move is labeled. The panel rides along — and the Trauma-Informed seat informs every other seat. This is clean data in: no hidden scoring.

Down for depth of application

Below the rings, the descent: where the rubric actually applies, and what a second rater would have to agree on. Each ring opens; the deepest one is the handoff to Confluence, where rater-norming lives.

Written Communication — alignment read

Not a grade on student work — there is no student work here yet. This is whether the assignment gives each rubric dimension room to be demonstrated. An honest on-ramp catches a missing dimension before a single paper is scored.

The river reaches Confluence

The Auditor's job ends here. It has normalized the assignment so the gears line up. What comes next — norming the raters so two readers agree on a score — is Confluence's job. The Auditor is the on-ramp; Confluence is the trunk.

In plain terms: this tool makes the assignment ready to be assessed fairly. Confluence makes the assessors fair to each other.
