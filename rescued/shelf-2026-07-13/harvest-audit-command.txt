HARVEST AUDIT
Drop-in command. Paste into any stale or to-be-archived Claude Project to
recover salvageable work before closing it.

When I write "Run harvest audit," do the following in order. Do not skip the
inventory step even if the project looks empty — orphaned work is invisible
until checked.

1. INVENTORY.
   List what exists as actual files on disk (/mnt/project/, outputs directory)
   versus what lives only in conversation history, memory summaries, or
   rendered-but-unsaved artifacts. Use the file tools to verify — do not assume.

2. SIGNAL VS. SCAFFOLD.
   Sort content into durable work worth keeping (finished tools, decisions,
   drafts, code) versus throwaway scaffolding (debugging chatter, abandoned
   attempts, superseded versions).

3. FLAG ORPHANS BY NAME.
   Identify every rendered artifact, pasted code block, or decision that exists
   only in chat and is not file-backed. These vanish on archive. Name each one
   specifically — no summaries like "various artifacts."

4. OFFER REGENERATION.
   Before packaging or closing, offer to regenerate each orphan into a real
   file in the outputs directory.

5. MANIFEST.txt.
   Produce a plain-text manifest: project name, harvest date, what's kept,
   what's dropped, known omissions.

6. DISPOSITION.
   Recommend one of: keep as-is, harvest-then-archive, or safe to delete.
   Give one default recommendation with a one-line reason — not a menu.
