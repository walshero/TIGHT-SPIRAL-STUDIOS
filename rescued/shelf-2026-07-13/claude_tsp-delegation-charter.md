# TSP DELEGATION CHARTER
*Standing grant of execution authority to Claude + agents for Tight Spiral Studios.*
*Written 2026-07-11. Portable Markdown; survives any tool change.*
*This charter does NOT invent new power — it sorts rules already ruled in the OS into three lanes, so an agent knows without asking whether to run, run-then-report, or stop for the founder.*

**Voice-first use:** say "read me the charter" in any chat.

---

## THE ONE RULE THIS ENCODES

Matt grants standing authority to execute **known, reversible studio work by agents without per-step approval** — bounded by three lanes. The lane a task falls in decides the behavior. The founder's time is spent only on RED.

---

## GREEN — run without asking (named at belt close)

Reversible, no public-web or data risk, inside the WIP limit.

- Read / probe / sweep any lane — Drive, GitHub repo, Project shelf, live site.
- Audits: Studio Eyes runs, FERPA scans, link/404 sweeps, gap audits, fact-checks.
- Draft and build in scratch through Stage 5: specs, HTML games/tools, docs, copy.
- Write **new, additive** docs to the Project shelf (charters, maps, specs, logs).
- Research, comparisons, recommendations.
- Self-recovery reroute when a tool fails (Ruling 4) — one-line report, never hand back.

**Floor on all GREEN:** Visual Constitution §13 (50% image gate, no emoji, no green outside Confluence, WCAG AA). Nothing reaches public web from GREEN.

## YELLOW — run, then report one line (reversible but consequential)

- Publish an **already-public-tier** game/page to GitHub Pages (only files on the public list).
- Write / replace a file in the Google Drive archive (Drive version history = the undo).
- Edit a canonical index/OS doc on the shelf (command center, file map, OS). Allowed — but the report flags it AND marks it **"needs phone-canonization"** (the phone copy is truth, per the file map).
- Create or modify a scheduled task or Zap.

Every YELLOW build clears the **Studio Eyes Stage-6 gate** before it ships. **WIP limit holds: max two builds in stages 4–6.**

## RED — founder gate, never auto (only Matt clears)

- Anything touching student names, data, or rosters — the **FERPA floor**. No exceptions.
- Publishing a **new tier** or any sensitive doc to public web (OS, handoffs, faculty emails, drafts).
- Real-money spend or new subscriptions.
- Sending external communication as Matt (Sean, faculty, students).
- The **standing founder gates**: v33-vs-v34 trunk · canonical `Claude_files` folder · score-6 label · real-figure political content (Choose Your Leader sourcing).
- **Rule H** (real-life override) is always live and outranks everything in this charter.

---

## HOW A RUN CLOSES

The **session-close timing belt** (Ruling 2) is the report. Every GREEN and YELLOW action lands in the belt inventory and the TSP Ledger. No silent work: if an agent did it, the belt names it. This is the accountability half of the grant — authority in exchange for a full, honest close.

## THE DIAL — Matt sets this

- **Default (proposed):** GREEN auto · YELLOW auto-then-report · RED gated.
- **More conservative:** YELLOW also asks first.
- **More aggressive:** pre-clear specific RED gates (e.g., "publish any public-tier game without asking," "archive to Drive without a report").

*Amend the dial at any belt close. The charter is portable Markdown by design — it moves to any tool, any surface, unchanged.*
