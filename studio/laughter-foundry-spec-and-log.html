The game — Busytown + Labyrinth + Nunan TBLT

Three influences, load-bearing

  • Lure of the Labyrinth. Standards live inside the puzzle. The machine will not run until parts mesh, and meshing them is the math/reading. No child sees a "problem"; they see a stuck contraption. Struggle is the assessment data.
  • Richard Scarry / Busytown. A dense, labeled cutaway world. Every part is named, captioned, doing a visible job. The catalog is a browseable page (the studio's visual-rhetoric and mnemonic standard).
  • Nunan, task-based language teaching. Pre-task, task cycle, language focus. Build the machine (task), interview the tester (report-back), reflect on what landed and why (language focus / metacognition).

The task cycle, four phases

  • Browse (pre-task). The cutaway Foundry catalog. Three named laugh-parts, tier-2 vocabulary. Line-art, single weight, legible small. No assessment yet.
  • Build (the task). Assemble a working chain. Gears mesh only at correct ratios; a release is timed by fraction of track; the run-timer is set in intervals. Wrong math means it will not run — the Labyrinth invisible gate.
  • Test and interview (report-back). Run it on a real tester; conduct the adapted Confluence interview. The tester quotes the moment; the player logs the response and writes a reason.
  • Reflect (language focus). Read the response bar graph; write an opinion-with-reasons about what to change. Metacognition, never graded to the child.

Two views

  • Kid view. The Busytown world, named parts, self-progress as parts mastered. No scores, no standard codes, no Not-Yet/Meets language. Just: this machine ran, this part is yours now.
  • Teacher view. Gated toggle. Evidence map: standard code, the quoted evidence span the child produced, the proficiency marker. Unmapped standards shown honestly — the Cartographer principle: name what is not yet demonstrated.
