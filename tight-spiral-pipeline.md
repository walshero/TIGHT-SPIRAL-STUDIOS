## The shape every stage shares

Every stage runs the same internal move:

> **agent pre-flight** (mechanical, binary — prepares the judgment) → **human gate** (the founder, the only judgment seat) → **emit**

The agent never judges. It clears the mechanical so the founder's scarce attention lands only on meaning. This is human-in-the-loop *operating*, not bending — the human is principal at every validation point.

Grounded in the studio's own sources (Gee, Nunan, Osterweil, AAC&U, the Confluence Measurement OS) and the founder's operating profile, not generic production orthodoxy.

---

## The five gate outcomes

Two outcomes (go/stop) are too blunt. Every gate emits exactly one of five:

| Outcome | Meaning |
|---|---|
| **GO** | Meets the stage's written exit criterion. Advance. |
| **CONDITIONAL GO** | Advance now on one named fix that must land before the next gate. The condition is logged; the next gate checks it first. |
| **RECYCLE** | Send back *one stage* — a specific upstream stage wasn't actually cleared. The idea is alive. |
| **HOLD / PARK** | Real, but not now. Leaves the pipeline for the parking lot with its un-park condition. |
| **KILL** | Stop. Either eliminate, or a fundamental flaw a later stage exposed. |
| **PIVOT** | *The spine changed.* A halt any stage (or the founder mid-build) can trip the instant the one-sentence answer to "what is this?" changes. Drops to a one-screen re-spine (§ below) before anything else proceeds. |

## The stages — define cheap (−1 to 3), ship minimal (4), iterate (5–6)

### −1. Intake Triage (the front-door gate)
*Mandatory. No build skips it.* Every incoming idea sorts one of four ways:
- **Build now** — moves institutional goals, cuts recurring load, or only the founder can do it, *and* there's attention for it. A "build now" that can't name what it displaces is a parking-lot item in disguise.
- **Park** — real, but not now. One-line reason + un-park condition. (Most ideas land here; that's healthy.)
- **Delegate to AI** — mechanical, spec-able, no judgment. Never enters the founder pipeline.
- **Eliminate** — exists only because a process is heavy. Cut before it costs a panel-minute.

Three questions in order: *Does this move an institutional goal or cut recurring load? Is there attention for it now, or must something yield? Who actually does it — founder, AI, or no one?*

### 0. Medium & Novelty
Run the **Medium Gate** (three lanes, no default: licensed photo/audio · raster/Lumino · cut-paper SVG) and name the one structural surprise that lives in the bones, not the paint. No named novelty → back it goes.

### 0.5. Convene the Disciplinary Panel
*The studio's signature move; do not skip.* Name the discipline and seat the minds who theorized it (4–8, never a library). Generative, not critical: it tells the game what it must *be* to honor its subject. Name a **spine**, an **entry/pacing** conscience, a **loop** conscience, a **core-mechanic** conscience.

### Feeling-first on the entry
The emotional entry — *does the first screen make the founder lean in, or reach for the close button* — is judged **before** the mechanic behind it is built. A 30-second scene mock, the founder's eyes, go/no-go.

### 1. Construct
Define what the build measures, *before any content*. State the evidence model; prefer computed evidence over self-report. Skipping this is the root cause of drift.

### 2. Task spine + Freedoms
The player does a discipline-real task (Nunan; rank by transfer: Roles > Move-Things > Combo > Puzzles). Locate the Four Freedoms; freedom to fail is non-negotiable for a noticing game.

### 3. Spec + Fidelity checklist
Lock parameters, draw the wireframe, and **sign the Fidelity checklist** the agent runs at every playtest pre-flight (construct intact, evidence on the move, freedoms present, accessibility floor, Visibility floor, scene-first floor, no-jargon-on-surface gate, house voice). Signed before the heat.

### 3.5. The Art Lane (when the Medium Gate chose photographic/generated)
Direct → generate (Firefly only; OS §16 license wall) → refinish (floors live here) →
embed (base64, ≤400KB/room) → gate the COMPOSITE through Studio Eyes. The raw generation
never ships. MJ never enters a shipping lane.

**Documentary / diagnostic imagery (the trainee reads it — smoke, fire, collapse):** source is
**real / public-domain, never generated** — fabricated fire teaches the wrong read (Registrar +
Hand seats HALT it). Ingest path when the sandbox can't reach the source: `WebSearch` to verify →
Zapier Upload File from URL → Drive → base64-embed, credited (`Madrzykowski/NIST · public domain`,
or the responder's dept). See `LANE-REGISTRY.md § THE ASSET-INGEST LANE`.

### 4. MVP build (the hinge)
Build the smallest thing that does the named job. Mechanic Prototype: art minimal, allowed to look plain. Defeats optimization-before-implementation — nothing downstream runs until the minimum ships.

### 5. Playtest loop (two ledgers) — the drift-control engine
- **Ledger A — Fidelity** (binary, unforgiving; a miss is a HALT)
- **Ledger B — Emergence** (captured, never scored; routes to the parking lot)

Inner loop: agent pre-flight (CLEAN/HALT) → founder plays once holding two questions (*does it still do its job? did anything surprise me?*) → **the Drift Fork**: *does this surprise serve the objective, or audition to replace it?* Serves → fold in; replaces → park it, named. → Calibration ping (did the measurement behave?).

### 6. Full work-up (Paper Craft)
Once the mechanic is proven, graduate to ship quality. Must pass C1–C5 asset rules. Gated *behind* a proven mechanic — never run on an unproven loop.

### 7. Ship & Canonize — including the HARVEST
Every build, before canonization, names:
- **The one thing it proved** (the harvest — promote a reusable asset).
- **The one thing it got wrong that the system should now catch** (the *Harvest-back* — new, 2026-06-30). This defect auto-runs a Probe Sweep (does this failure-class exist elsewhere?) and queues the fix promote-or-kill in the Recursion Ledger.
- Retire superseded files. Twin rule: adding a rule prunes one.

## The re-spine (what PIVOT drops to)
*One screen, before any build.* Three lines, no more:
1. **New spine** — the new one-sentence "what is this game," written next to the old one.
2. **What it displaces** — replaces the old game, or sits beside it. Name which.
3. **What survives** — which decided calls, sourced content, and assets carry over, and which are now void.

Then re-enter the pipeline, usually at Stage 0.5 (the discipline may have changed).

## The recursion loop (governance — locked 2026-06-30)
The pipeline corrects itself from its own output, automatically below a wall it cannot cross.

> Ship → Stage-7 Harvest-back names what broke → auto Probe Sweep finds the failure-class across the system → propose a fix **calibrated to a named reference** (a floor, a usability heuristic, a decision-log entry, founder eyes, or a domain standard) → the fix is legal **only below the hard wall** (emoji / trauma / FERPA / child-safety can never recurse) → **best-practice-tier fixes graduate automatically**; hard-wall changes require a founder gate → **one graduates per build** (the rate governor) → the decay rule prunes a twin → re-ship.

The founder gate sits at "graduate" for hard-wall items only; below the wall, calibration + the one-per-build cap are the control. The executable form is `recursion-ledger.html`.

## The three playbook gates (run at parameter time, before build)
- **Wireframe Novelty Gate** — name the one structural surprise.
- **Medium Gate** — three lanes, no default; never grind one medium toward what another does.
- **Panel-Selection Gate** — convene only the seats the work touches.

## The Tight Spiral loop (how a review resolves)
Diagnose, fix, re-test, *and fix sibling issues* — one reported symptom usually has relatives. The loop is the studio's name and its method.

## Convening logic
Studio Build = Craft seats. Classroom Deliverable = Craft + Classroom. Governance = all three. Keep the panel as small as the job allows.

## The Ten Questions (universal review)
1. What assumptions are we making?
2. What evidence supports them? (observable, not asserted)
3. How could we be wrong?
4. What can be deleted?
5. What can become simpler?
6. What should become reusable?
7. How will we measure success? (the construct, defined before content)
8. How will we detect failure? (the two-ledger playtest)
9. How will the user understand the system? (label the move)
10. What did we learn? (the durable signal that outlives this build)
