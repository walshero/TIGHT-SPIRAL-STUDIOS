# Three-Lane AI Map — Tight Spiral Studios
*walshero@gmail.com · the one-page reference. Set it up once, follow it later.*
*Core law: each AI owns ONE lane. No AI is the record. The FILES are. Everything drains to one shelf.*

---

## THE ONE TRUE SHELF
**Google Drive `Claude_files`** is the single source of truth. Every lane ends by saving a file here. The Project (in Claude) is the working face that auto-loads. The zip is a backup, kept OUTSIDE the Project. Save from the PHONE — one device, one gesture (kills the collision-copy problem).

---

## LANE 1 — CLAUDE = HOME for Tight Spiral work
**Owns:** the builds (games, tools), the OS and its law, the studio voice, every design decision, file production (HTML/SVG), the panel/Conductor.
**Does NOT:** do open-ended web research libraries, touch real student data, replace your hand on relationship/creative content.
**Carries info back to Claude:** files in the Project. That IS the only real hook — there is no live channel from a website or another AI into Claude. Manual-but-reliable.

---

## LANE 2 — CHATGPT PRO = pipeline & research (the Scriptorium)
**Owns:** research corpora, standards crosswalks, government/public-data pulls, Eagle-Eye monitoring, concept-art direction that feeds Midjourney. THINK-before-MAKE.
**Does NOT:** build the games, write OS law, make design calls, author in your voice.

**Setup (once):**
1. Left sidebar → **+ New Project** → name it "Tight Spiral — Scriptorium".
2. Upload the three Scriptorium templates (job-card, panel-packet, agent-pipeline doc) — they become its manual.
3. Project → **Instructions**, paste:
   > "You are the Scriptorium — the research and pipeline wing of Tight Spiral Studios. Every job starts as a Job Card. You gather current research, standards, and public data; you tag findings METHOD / STANDARD / DOOR; you never make design decisions (that's Claude's lane). Every output ends as a file I save to Google Drive Claude_files. Cite sources. Flag freshness risk. Do not author creative or relationship content."
4. Use **Deep Research** mode for corpus-building (the federal-data libraries, standards pulls) — that's what Pro is worth paying for.

**Hand-off:** ChatGPT makes a corpus/brief → you save it as a file → into Drive → upload to the Claude Project when a build needs it.

---

## LANE 3 — GEMINI = assessment structure (FERPA-walled, narrow)
**Why Gemini, honestly:** not "better at assessment" — it lives INSIDE Google Workspace where your assessment artifacts already are (Sheets/Docs), which cuts the copy-paste that RP makes costly.
**Owns:** norming/calibration math, rubric crosswalks, ISLO-mapping across the six levels, turning rubric language into readable reports — on DE-IDENTIFIED material only.
**Does NOT:** make the validity call on any instrument (that's the consulted-expertise ship gate), or touch anything with a named student.

**Setup / use:**
1. Use Gemini **inside Docs and Sheets** (the "Ask Gemini" side panel, right edge of the document) — NOT the standalone app. Keeps data in Drive, never pasted into a chat box.
2. Safe inputs: the placement rubric, AAC&U rubrics, the calibration framework (no student data).
3. **HARD FERPA RULE:** never put real student names or identifiable work into ANY AI. De-identify first.

**Hand-off:** Gemini outputs live in Drive Docs/Sheets → bring only the de-identified summary into the Claude Project when it should shape a build (e.g., Confluence).

## MIDJOURNEY (concept art, when raster is needed)
No live plug-in to Claude — and that's fine. The honest pipeline (already in your Scriptorium doc):
**ChatGPT Pro / Midjourney generate concept art → you save the image → Claude builds the production asset from it.** A human-in-the-loop handoff, not a hook.

## THE FLOW, ONE LINE EACH
- ChatGPT Pro → research/pipeline → saves to Drive → feeds Claude.
- Gemini → de-identified assessment structure, inside Workspace → stays in Drive.
- Claude → builds, OS, voice, design → home for Tight Spiral work.
- Drive `Claude_files` → the one shelf. Every lane drains here. No AI is the record.

## RP / FRICTION GUARDS
- Keep each AI in its own browser profile or tab group — don't hunt for which window is which.
- Phone-save carries everything to the shelf — the move that makes three AIs safe instead of chaotic.
- Calendar: don't build a new one. Link out to the Google/Outlook calendar you already run by voice.

## THE DISCIPLINE
Three AIs multiplies the source-of-truth risk you fixed this session. The lane law is what keeps it safe: one owner per lane, one shelf for all. When in doubt, the file on the shelf wins over anything any AI remembers.
